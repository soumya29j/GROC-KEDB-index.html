KEDB-01: Article is not the same for confirmation and delivery

FieldDetails
ErrorArticle is not the same for confirmation and delivery
IDoc0000000485232182
Message TypeSHP_OBDLV_CREATE_STO
SiteSP9K

Observation

Delivery creation IDoc 0000000485232181 was processed successfully (Status 53). The payload contained both consumable and non-consumable articles and generated the message: You cannot create several delivery headers in case of external number assignment.

Root Cause

SAP requires separate deliveries for consumable and non-consumable articles. Since both article types were transmitted in a single payload, SAP created the delivery only for the non-consumable article. During PGI confirmation, the system received confirmation data for an article that was not available in the delivery, resulting in the error.

Resolution

  • Maintain Route ID in the consumable STO.
  • Delete the existing delivery from Shipsy.
  • Re-push the request from Shipsy.
  • Verify successful delivery creation in SAP.

Conclusion

The issue occurred because consumable and non-consumable articles were sent together in a single payload. Maintaining Route ID and recreating the delivery resolves the issue.

KEDB-02: Order item already delivered or not delivery relevant

FieldDetails
ErrorOrder item 4780573570 / 000003 already delivered or not delivery relevant
IDoc0000000485099425
Message TypeSHP_OBDLV_CREATE_STO
SiteSP9K

Root Cause

The Delivery Completion Indicator was automatically set for STO line item 000003 on 31.05.2026 through process OPS_PO_CLS. SAP therefore considered the item completed and prevented delivery creation.

Resolution

  • Remove the Delivery Completion Indicator from the STO line item.
  • Wait for the standard reprocessing cycle (~30 minutes).
  • Reprocess the IDoc.
  • Validate successful delivery creation.

Conclusion

The issue was caused by the Delivery Completion Indicator being set on the STO item. Removing the indicator resolved the problem.

KEDB-03: Deficit of Delivered Block Stock Quantity

FieldDetails
ErrorDeficit of Delivered blStck qty 2 CAR : 494354706 SP9K
Failed IDoc0000000485476123
Successful IDoc0000000485596581

Issue Details

The document posting IDoc failed because mandatory fields were not populated in the incoming payload.

Missing Fields

  • Header Text
  • REF_DOC_YR
  • REF_DOC_IT

Root Cause

The initial IDoc was received without the mandatory reference fields. SAP validation failed during processing. When the same transaction was resent with the required values populated, the document posted successfully.

Resolution

  • Verify that Header Text, REF_DOC_YR and REF_DOC_IT are populated before transmission.
  • Resend the transaction with complete data.
  • Monitor successful IDoc processing.

Conclusion

Missing mandatory reference fields caused the posting failure. Resending the payload with complete information resolves the issue.

KEDB-04: Delivered Block Stock Quantity Exceeded

FieldDetails
ErrorDelivered blStck qty exceeded by 9 C04 : 491161238 SP9K 3020
Failed IDoc0000000485596610
Successful IDoc0000000485509943

Observation

Movement 105 had already been posted against movement 103. A duplicate re-push from Shipsy triggered another IDoc, resulting in the quantity exceeded error.

Root Cause

The original transaction was already processed successfully in SAP. A subsequent duplicate re-push attempted to post the same stock movement again, causing SAP quantity validation to fail.

Resolution

  • Do not reprocess these IDocs if the original posting was successful.
  • Validate document status before re-pushing transactions.
  • Raise the issue with Shipsy to identify the reason for duplicate re-push requests.

Conclusion

This is a duplicate processing scenario. No SAP correction is required if the original document has already been posted successfully.
